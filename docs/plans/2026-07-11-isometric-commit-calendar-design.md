# Isometric Commit Calendar Design

## Goal

Create the initial public GitHub Profile README for `allenchenhan99` and display a full-year isometric commit calendar generated by `lowlighter/metrics`.

## Chosen approach

Use GitHub's native profile repository mechanism: a public repository named `allenchenhan99/allenchenhan99` whose root `README.md` appears at the top of the user's Overview. A scheduled GitHub Actions workflow will run `lowlighter/metrics@latest`, generate `metrics.plugin.isocalendar.svg`, and commit the rendered asset back to the profile repository.

This approach was chosen over the shared Metrics instance because it supports the full plugin configuration and avoids depending on the shared renderer's availability. Self-hosting or maintaining a fork is intentionally deferred because the initial calendar only needs the Metrics isocalendar plugin and a least-privilege token.

## Profile presentation

- Show the full previous year of contribution activity using `plugin_isocalendar_duration: full-year`.
- Disable base metrics with `base: ""` so the first iteration contains only the isometric calendar.
- Center the generated SVG in the README and give it meaningful alternative text.
- Do not add banners, skill badges, statistics cards, or marketing copy in this iteration. Those will be considered in the later professional-profile design discussion.

## Automation and data flow

1. GitHub Actions runs daily, on manual dispatch, and after workflow changes land on `main`.
2. The workflow passes the repository secret `METRICS_TOKEN` to `lowlighter/metrics@latest`.
3. Metrics queries GitHub contribution data for `allenchenhan99` and renders `metrics.plugin.isocalendar.svg`.
4. The Action commits the generated SVG to the profile repository using `contents: write` permission.
5. `README.md` displays the repository-local SVG on the public Overview.

## Private contributions and security

- Include private contribution counts when GitHub's "Include private contributions on my profile" setting permits it.
- Never expose private repository names or contents in the README or generated asset.
- Store the personal access token only as the encrypted repository secret `METRICS_TOKEN`; never place it in files, command output, commits, or chat.
- Use the least privilege supported by Metrics for this plugin. Revisit a controlled fork or stronger version pinning before adding plugins that require broader scopes.

## Failure handling

- Keep `workflow_dispatch` enabled so a failed scheduled run can be retried manually.
- Validate the workflow syntax locally before publishing.
- After publishing, confirm the first Actions run succeeds, the SVG exists, and the public profile renders the image.
- If the Metrics render fails because the token is absent or insufficient, do not broaden permissions silently; report the exact missing prerequisite.

## Success criteria

- `https://github.com/allenchenhan99` displays a full-year isometric contribution calendar above the native pinned repositories.
- The generated calendar includes eligible private contribution counts without revealing private project identities.
- The workflow updates daily and supports manual reruns.
- No credential is present in repository history or workflow logs.
